SECTION A – ORIGINAL PROJECT INTENT (HISTORICAL TRUTH)
================================================================================

GFC Web Studio was intended to be a **visual content and animation system**
integrated into the broader GFC Ecosystem. Its core purposes are:

- Provide a modern, visual way to build and edit pages
- Host advanced animations originally developed in Animation Playground
- Allow non-developers (admins) to manage visual content safely
- Eventually replace ad‑hoc hard‑coded UI/animation changes
- Remain logically separate from business logic (membership, access control)

Studio is NOT:
- A replacement for the GFC Web App business logic
- A controller management tool
- A generic CMS without guardrails

[R2-1] LAUNCHED FROM WEB APP
---------------------------
- Studio is accessed from within the GFC Web App.
- Identity and permissions are inherited.
- Standalone access is not assumed unless explicitly added later.

[R2-2] DEEP LINK & CONTEXT LAUNCH
--------------------------------
The Web App may launch Studio with context:
- Edit specific page
- Edit navigation/footer
- Edit rentals page content
- Edit global theme/settings

Permissions are validated before opening.

[R2-3] CONTENT VALIDATION RULES
-------------------------------
Studio enforces validation before publish:
- Required fields present
- Text length bounds
- Image size/aspect ratio checks
- Animation parameter min/max bounds

[R2-4] DRAFT / PREVIEW / PUBLISH LIFECYCLE
-----------------------------------------
- Draft edits do not affect public site.
- Preview mode exists (mechanism to be defined).
- Publish creates a revision snapshot.
- Rollback restores last-known-good revision.

[R2-5] VERSION & SCHEMA COMPATIBILITY
------------------------------------
- Studio schemas must remain compatible with website renderer.
- Breaking changes require version bump and documented migration.
- Silent breaking changes are not allowed.

[R2-6] EXPLICIT NON-GOALS
------------------------
Studio will NOT:
- Manage members, dues, or access control
- Process payments
- Bypass Web App permissions or audit
- Write directly to Web App databases

[R2-7] SINGLE SOURCE OF TRUTH (GOVERNANCE RULE)
----------------------------------------------
- Studio is the single source of truth for website content.
- Web App controls permissions, audit, and publishing authority.
