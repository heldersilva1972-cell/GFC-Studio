SECTION F – CONTENT IMPORT AND MIGRATION
STATUS: UNDER REVIEW
================================================================================

F.1 PURPOSE
STATUS: PENDING
The Content Import and Migration system exists to safely and deliberately migrate
content from legacy sources (such as the existing GFC website or external HTML
sources) into Studio’s structured block-based model. This system prioritizes
correctness, traceability, and editor control over automation.

Imported content MUST NEVER be published automatically. All imported content
enters the system as Draft with explicit import markers and mandatory review.

--------------------------------------------------------------------------------
F.2 SCOPE
STATUS: PENDING

F.2.1 Included
STATUS: PENDING
• One-time and manual content imports
• Legacy HTML and text parsing
• Media reference extraction
• HTML-to-block conversion
• Import tagging and lineage tracking
• Mandatory post-import review workflow

F.2.2 Excluded
STATUS: PENDING
• Continuous sync with external sites
• Auto-publish or background imports
• Execution of imported scripts or styles
• Automated content rewriting

F.3 IMPORT SOURCE DEFINITION
STATUS: PENDING

F.3.1 Supported Source Types
STATUS: PENDING
The import system MUST explicitly declare supported sources, including:
• Static HTML files
• URL-fetched HTML (admin-only)
• Plain text documents

Unsupported source types MUST be rejected.

F.3.2 Source Validation
STATUS: PENDING
Before import begins, the system MUST:
• Validate source accessibility
• Confirm source size limits
• Sanitize input to remove scripts, iframes, and inline styles
