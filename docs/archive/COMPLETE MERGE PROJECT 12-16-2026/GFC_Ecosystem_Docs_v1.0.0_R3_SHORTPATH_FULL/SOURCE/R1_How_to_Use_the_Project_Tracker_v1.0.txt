
==================================================
HOW TO USE THE PROJECT TRACKER (ONE-PAGE GUIDE)
VERSION: 1.0
AUDIENCE: ALL USERS / ALL CHATS
==================================================

PURPOSE
This guide defines the exact, repeatable process for using the Project
Tracker so that work continues correctly across chats, users, and time.

--------------------------------------------------
STARTING A NEW CHAT
--------------------------------------------------

YOU PROVIDE:
- Project name
- Active CIR ID
- Latest Cassette OR document ZIP

Example:
Project: Project Tracker
Active CIR: CIR-PT-001
Documents: (attached)

AI WILL:
- Confirm documents loaded
- Confirm governance active
- List completed, blocked, and next available steps

--------------------------------------------------
CHOOSING WHAT TO DO
--------------------------------------------------

YOU SAY ONE OF:
- Continue execution
- Proceed with STEP X
- Design UI for STEP X
- Resolve BLOCK in STEP Y
- Add new feature (creates new CIR)

AI WILL:
- Identify affected step(s)
- Confirm if execution is allowed
- State which document will change next

--------------------------------------------------
EXECUTION
--------------------------------------------------

YOU SAY:
Proceed with STEP X

AI WILL:
- Provide ONE prompt
- Include Prompt ID and Revision
- Define exact execution scope

--------------------------------------------------
CONFIRMATION & PROOF
--------------------------------------------------

YOU PROVIDE:
- “Prompt completed” OR
- Upload logs / screenshots / output OR
- Error details

AI WILL:
- Create Execution Record
- Create Completion Record OR Fix/Block Record
- Update step status

--------------------------------------------------
SNAPSHOT (OPTIONAL)
--------------------------------------------------

YOU SAY:
Generate Cassette

AI WILL:
- Produce a frozen, auditable project snapshot

--------------------------------------------------
CORE RULES (NON-NEGOTIABLE)
--------------------------------------------------

- Intent lives in documents, not chat
- AI never guesses
- Completed steps are never re-run
- No step is complete without proof
- Governance always overrides conversation

==================================================
END — HOW TO USE THE PROJECT TRACKER
