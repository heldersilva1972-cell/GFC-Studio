SECTION D – SANDBOX AND SAFE EXPERIMENT MODE
STATUS: UNDER REVIEW
================================================================================

D.1 PURPOSE
STATUS: PENDING
Sandbox and Safe Experiment Mode provides a fully isolated environment for
testing, experimentation, and iteration without risk to active drafts or
published content. This mode allows editors to explore layout changes,
animations, and content variations while preserving the integrity of the
primary editing workflow.

Sandbox behavior is explicit, isolated, auditable, and time-bound.

--------------------------------------------------------------------------------
D.2 SCOPE
STATUS: PENDING

D.2.1 Included
STATUS: PENDING
• Per-page sandbox creation
• Full isolation from drafts and published content
• Authenticated preview access
• Promotion of sandbox to draft
• Sandbox expiration and cleanup

D.2.2 Excluded
STATUS: PENDING
• Public indexing or visibility
• Direct publish from sandbox
• Background or automatic promotion

D.4 SANDBOX CREATION
STATUS: PENDING

D.4.1 Trigger Conditions
STATUS: PENDING
A sandbox MAY be created when:
• User has edit permission
• Source page exists in Draft or Published state

D.4.2 Creation Mechanics
STATUS: PENDING
Sandbox creation MUST:
• Clone content and animations
• Assign new SandboxId
• Record lineage to source revision
• Initialize validation state

D.4.3 Failure Handling
STATUS: PENDING
If sandbox creation fails:
• No sandbox record may persist
• User MUST be notified

STATUS: PENDING
